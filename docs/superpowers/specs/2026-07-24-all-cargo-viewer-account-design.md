# 전체 화물 게스트 조회 계정 설계

## 목표

`guest` 계정은 특정 화주 필터에 묶이지 않고 모든 화주의 화물을 조회한다. 화면은 화주용 `BL별 진행현황`을 사용하지만, 관리자 기능과 모든 데이터 변경 기능은 사용할 수 없는 읽기 전용 계정이다.

## 권한 모델

- 기존 `admin`: 전체 화물 조회 및 관리 기능 유지
- 기존 `shipper`: 계정에 연결된 화주의 화물만 조회하고 허용된 화주 입력 기능 유지
- 신규 `viewer`: 전체 화물 조회만 허용

`viewer`는 화면에서 버튼을 숨기는 것에 그치지 않고 서버 API에서도 모든 쓰기 요청을 거절한다.

## 데이터베이스

`shipper_accounts.role` 허용값에 `viewer`를 추가한다. 계정 관리 RPC도 `viewer` 역할을 보존하도록 확장한다.

운영 계정은 다음 값으로 생성한다.

- 로그인 아이디: `guest`
- 비밀번호: 사용자 지정값
- 표시명: `전체 게스트`
- 역할: `viewer`
- 화주 필터: 빈 값
- 활성 상태: 사용

마이그레이션은 재실행해도 안전한 형태로 작성한다.

## 조회 동작

`GET /api/cargo-data`는 역할별로 다음 범위를 반환한다.

- `admin`, `viewer`: 모든 `cargo_cards`와 연결된 사용자 입력, 요청, 원본서류 상태
- `shipper`: 세션의 `account_id`에 해당하는 자료만 반환

응답의 사용자 역할은 그대로 `viewer`로 전달하여 프런트엔드가 읽기 전용 화면을 구성하게 한다.

## 쓰기 차단

공통 인증 단계에서 `viewer` 세션의 `POST`, `PUT`, `PATCH`, `DELETE` 요청을 거절한다. 로그인·로그아웃과 조회 API는 허용한다.

추가로 프런트엔드에서 다음 요소를 표시하지 않거나 비활성화한다.

- 관리자 설정, 숨김 관리, 관리자 상태 토글
- 서류수령·수입신고·출고 요청
- 운송정보 입력
- 확인사항 추가·수정·삭제
- 할당관세 및 원본서류 상태 변경

`viewer`는 기본 화면인 `BL별 진행현황`과 캘린더를 조회할 수 있다.

## 계정 관리

관리자 설정의 권한 선택에 `전체조회(읽기 전용)`을 추가한다. `viewer` 계정은 필터텍스트가 없어도 저장할 수 있다. 기존 `shipper` 계정의 필터텍스트 필수 검증은 유지한다.

## 검증

- 역할별 데이터 조회 범위 테스트
- `viewer` 쓰기 API 거절 테스트
- 관리자 설정의 `viewer` 계정 저장 테스트
- `viewer` 화면에서 변경 컨트롤이 노출되지 않는 테스트
- 운영 DB 마이그레이션 및 `guest` 계정 생성 확인
- 실제 `guest` 로그인 후 전체 화물 수와 읽기 전용 UI 확인
