# 유통이력 번호 호버 및 양도증 캘린더 정리 설계

## 목표

- BL별 진행현황 하단 캘린더에서 양도증 수령 이벤트만 제외한다.
- 유통이력 O 배지에서 저장된 본인·본레스 신고번호를 빠르게 확인하고 복사할 수 있게 한다.

## 범위

### 양도증

- 양도증 파일 감지, O/X 상태, 관리자·모바일 관리 기능은 유지한다.
- 진행현황 표의 양도증 열도 유지한다.
- 캘린더 이벤트 생성 단계에서 양도증 수령 이벤트만 만들지 않는다.
- OBL·H/C 원본 수령 이벤트와 다른 일정은 변경하지 않는다.

### 유통이력

- 관리자가 유통이력을 X에서 O로 변경할 때 기존 본인·본레스 입력 모달을 사용한다.
- 본인 또는 본레스 신고번호를 하나 이상 입력한 경우에만 O로 저장한다.
- O 배지를 호버하거나 키보드로 포커스하면 저장된 번호를 표시한다.
- 팝업은 배지와 같은 호버 영역에 포함하여 마우스를 팝업으로 이동해도 유지한다.
- 팝업 텍스트에는 `user-select: text`를 적용해 드래그 복사가 가능하게 한다.
- 번호가 없는 기존 O 데이터는 O 배지만 표시하고 빈 팝업은 만들지 않는다.

## 데이터와 API

- 기존 `distribution_history_override`와 `distribution_history_number` 필드를 그대로 사용한다.
- API와 Supabase 스키마는 변경하지 않는다.
- 번호 형식은 기존 `본인:<번호> / 본레스:<번호>` 형식을 유지한다.

## 검증

- 양도증 수령 상태가 O여도 캘린더에 양도증 이벤트가 생성되지 않는지 확인한다.
- OBL·H/C 수령 이벤트는 계속 생성되는지 확인한다.
- 유통이력 번호가 있으면 O 배지와 선택 가능한 팝업이 렌더링되는지 확인한다.
- 본인 또는 본레스 번호가 모두 비어 있으면 X에서 O로 저장되지 않는지 확인한다.
- 기존 홈페이지 전체 테스트와 로컬 미러 일치 테스트를 통과시킨다.
